## 🛠 How to use it

This toolkit is built around simple working steps:

1. Start with the project goal.
2. Break the work into small sprint tasks.
3. Let AI agents draft code and changes.
4. Review each result before it moves forward.
5. Keep the human role focused on direction, checks, and approval.
6. Repeat the cycle until the app is ready for release.

For a non-technical user, the main job is to follow the prompts, read each step, and confirm choices when asked. You do not need to know code to use the workflow.

## 📋 Typical workflow

A common use flow looks like this:

1. Open the toolkit.
2. Create or load a project plan.
3. Choose the sprint you want to work on.
4. Let the AI agent make the first draft.
5. Review the output.
6. Approve, revise, or send it back.
7. Move to the next task.

This keeps work organized and makes it easier to track progress across the project.
